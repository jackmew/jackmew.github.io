mvn help:effective-pom => 查詢現在 pom.xml 的有效設定

我根本就不用手動去下載log4j , 只要在pom.xml裡加上

        <dependency>
            <groupId>commons-logging</groupId>
            <artifactId>commons-logging</artifactId>
            <version>1.1.1</version>
        </dependency>
        <dependency>
            <groupId>log4j</groupId>
            <artifactId>log4j</artifactId>
            <version>1.2.16</version>
        </dependency>

下mvn package 後就會自動下載需要的jar

怎麼來的?

原本有加上apply plugin:’application’

所以可以直接gradle run

但改掉後 就沒辦了

build.gradle:

apply plugin: 'java'

repositories {
    mavenCentral()
}

dependencies {
    compile group: 'commons-logging', name: 'commons-logging', version: '1.1.1'
    compile group: 'log4j', name: 'log4j', version: '1.2.16'
}

sourceSets {
    main {
        java {
            srcDir 'src'
        }
        resources {
            srcDir 'config'
        }
    }
}

gradle build 沒有問題

但是java -classpath builds/lib/helloworld2.jar tw.com.codedata.App 出現lost logFactory的訊息

看一下helloworld2.jar 裡面 沒有包入commons-logging,也沒有包入log.class

需要再加入

jar {
    from configurations.compile.collect { zipTree it }
}

表示會在jar中加入dependecies

java -classpath builds/lib/helloworld2.jar tw.com.codedata.App
