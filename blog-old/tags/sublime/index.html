Node.js 事件迴圈

要達到Asynchronous I/O的特性,必要有一個監聽者的角色來處理非同步發出的請求,像callback開始後 要花一點時間完成 所以先執行了後面的程式,那誰來接收callback回來的結果呢,是因為有個監聽者一直在不斷的迴圈,看有沒有任何事件,請求,回應會突然出現,這監聽者的角色就是 libev

CommandJS

RequireJS

JavaScript本身就具備Require指令來載入模組,然而我們發現並不是所以模組都要在一開始就載入好的

RequireJs可以做到當下需要的模組再載入的這個想法也就是Asynchrnous Module
Defintion(AMD),並且確保所以相依(dependency)當載入才會執行

CommandJS和RequireJS有什麼不同

module & package

package比較像是組合一堆module之後的完成品

export : 公開模組

require : 取得模組

有點類似於java的class

//hello.js

function Hello() {
 var name;

 this.setName = function(thyName) {
   name = thyName;
 };

 this.sayHello = function() {
   console.log('Hello ' + name);
 };
};

module.exports = Hello;

有點類似於java new 實體

//gethello.js

var Hello = require('./hello');

hello = new Hello();
hello.setName('BYVoid');
hello.sayHello();

Javascript是函數式程式設計語言,並沒有class概念,function本身就是物件,在node又叫做module,module可以藉由export公開,require引用

package.json

CommonJS規定用來描述套件(package)的檔案

Node.js會先去找package.json裡面的main 決定要先執行哪個js 若是沒有package.json就會去找index.js or index.node
