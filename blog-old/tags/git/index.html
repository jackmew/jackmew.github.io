design

concept

  1. extends

A extends B

在一般的狀況下 A都會有default constructor 去呼叫B的constructor

public A(){
    super();
}

但是當你要自定義呼叫B的constructor時(constructor可以無參數 有參數 , 兩個參數) , default constructor就不會動作！ 因此若是你有定義一個參數的construtor , 你就應該去呼叫無參數的constructor ! 不然就會無呼叫 B的東西可能就會少

  1. vaadin UI

在建構vaadin ui component時 , 你可以在兩個地方建構 :

  1. constructor -> 當我們使用此component時 都會new , 因此都會呼叫到constructor.
  2. init -> constructor 之後 就會執行init , 這時候會發現一個問題 , 若是有些人在constructor時 就去呼叫別的class , 而此class可能又會去call其他class ,那我們要怎麼保證 class 已經construct 完成 ? init就像是$(“document”).ready 一樣 , 會在整個元件都準備好才載入 , 因此可以保證 , 所以我們的constructor儘量沒東西 ,都放在init裏面 .

GitHub : open source version control system(VCS)

一開始會知道這個,只是因為常常發現有很多網路上的教學或範例都給你個url,說範例在這,你自己來試試看.

….到底是要怎麼試啦!!

網路上已經有很多教學了,我覺得挺有用的是這篇blog.gogojimmy.net

當你要下載別人的範例,比如說我寫的一個簡單jquery plugin

cd 你要放的資料夾位置 下git clone xxxurl

github還有follow fork功能 , 讓你可以追蹤有興趣的code,甚至你還可以去幫別人改code
