##繼續解決jquery mobile enhance的問題

###jQuery Mobile’s Auto-initialization

link

jQuery Mobile markup enhancements的行為是 當被load完就會套上 在document.ready之前 .

mobileinit 當jQuery Mobile load完的時候,觸發此事件

但我無法觸發此事件…原因 : jquery.mobile-1.4.0” 要放在你引用的js之後

Order:

  1. jquery-1.10.2.js”
  2. UIDWidget.js”
  3. jquery.mobile-1.4.0.js”

然而這樣改變順序 會影響你之前設定的變化 也會影響onDocumentReady 和 page’s event的順序

###Working with jQuery Mobile’s Auto-initialization

link

###了解jQuery Mobile 流程

Understanding the Mobile Initialization Event

jQuery framework 用 $(document).ready() 去規避處理和讀取的問題 當成程式進入點用 ,這在sigle page sites很有用 但是跟jQuery Mobile framework搭在一起會有問題

jQuery Mobile 用 AJAX 的方式 讀取頁面的內容 ,不是reload整個page ,

$(document).ready() will trigger before your first page is loaded

Prevent page transition

##Soving problem

關鍵字 : jquery mobile requirejs

https://groups.google.com/forum/#!msg/requirejs/LUjOcKxERGU/Rphf0x0jPm4J

first : 我猜測 onDocument 之前還有很多page event 但不知道先後順序

second : jquery mobile的include 要在你的custome js 之後

改變順序後 onDocumentReady會再page event之前 但其實邏輯上本來就應該是這樣

首先load jquery , 再load你寫的js 你的js才可以用$ 同時load DOM , 當load完DOM時 才能去對DOM enhance 所以最後是load jquery mobile

third : requirejs的 非同步動態載入 配置

現在mobileinit已經可以call了 但是page event還是不行

mpc(mobile page container) It’s a code that should run after mobile init

Use jquery mobile with requirejs, delay pageinit function until scripts are loaded

possible conflict with Jquery Mobile “mobileinit” event

以上的種種 都是library,page的load先後順序問題,表示這個行為 應該是每一個頁面的共同行為

用requirejs定義共同load行為

‘weaver/domReady!

if (isBrowser) {
        $(document).bind( "mobileinit", function() {
            // Switch off autoinitialization of page
            $.mobile.autoInitializePage = false;
        });
        require(["weaver/jquery.mobile", "weaver/jquery.validate"], function(jqm, validator) {
            $(document).ready(function() {
                jqm.initializePage();
                pageLoaded();
                require(["weaver/jquery/localization/messages_zh_TW"]);
                $(document).show();
            });
        });
    }

##docReady!

HTML => document Ready
jquery => document Ready
requirejs => document Ready!

是一層層的關係 並不是同一個東西

RequireJS

  1. require表示要執行的程式

  2. define表示包成模組

entry point:

  1. data-main

  2. docReady!

display:none vs visibility:hidden

差別

//display:none =>會把height,width設成0 , 之後show可能會有點問題 => $(selector).show()

//visibility:hidden =>會佔住位置 , 之後show ok => //$(selector).css(‘visibility’,’visible’)

雖然最後解決的方法用最簡單的visibility:hidden //$(‘body’).css(‘visibility’,’visible’)
