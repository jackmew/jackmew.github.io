之前就發現 轉頁面的時候 會先顯示出沒有jquery mobile enhance 再隔一下跳視窗才會enhance
一直不想管這問題 好像很難解決..

猜測應該是pagebeforeshow or pageinit 先讓他enhance 再show出來這樣

算是精華篇吧!!! $(document).on(‘pageinit’) vs $(document).ready()

$(document).ready() will trigger before your first page is loaded and every code intended for page manipulation will be executed after a page refresh.

page init vs beforeshow

it is advised to separate all of your custom javascript page handling into a single separate js file.

Page Change Times

  1. A page change process is begun
  2. A new page is loaded
  3. The content for that page is “enhanced” (styled)
  4. A transition (slide/pop/etc) from the existing page to the new page occurs

我必須知道順序性才行

1.$(document).on(‘pagebeforechange’, function(e, data){
alert(“pagebeforechange”);
});
執行了兩次

2.$(document).on(‘pagecreate’, function(e, data){
alert(“pagecreate”);
});

3.$(document).on(‘pageinit’, function() {
alert(“pageinit”);
});

4.$(document).on(“pagebeforeshow”, function(){
alert(“pagebeforeshow”);
});

5.$(document).ready(alert(“ready”);
)

so the pageinit event provides the correct timing for post enhancement manipulation of the DOM and/or Javascript objects

所以應該要在pageinit就enhance
