###Model(store)

在之前的投影片裡是定義 Store 包含多個model 實體(instance)

=>MyApp/app/store/People.js

Ext.define('MyApp.store.People',{
    extend: 'Ext.data.Store',
    config:{
        model: 'MyApp.model.People',
        autoLoad: true,
        proxy:{
            type:'ajax',
            url:'/app/data/People.json',
            reader:{
                type: 'json',
                rootProperty: 'Person'
            }
        }
    }
});

包含一個model:’MyApp.model.People’

proxy定義去哪裡抓資料 url:/app/data/People.json

=>MyApp/app/model/People.js

Ext.define('SogetiApp.model.People',{
    extend: 'Ext.data.Model',
    config:{
        idProperty: 'Id',
        fields:[
            { name: 'Id', type: 'int'},
            { name: 'Name', type: 'string'},
            { name: 'Email', type: 'string'}
        ]
    }
});

定義抓出來的資料欄位 格式

轉到第三頁,MyApp/app/view/Address.js

extend: ‘Ext.List’ 表示Address繼承了List componet

比較特別的是它去model取得資料,並定義了呈現資料的方式

store: Ext.create(‘SogetiApp.store.People’), 去MyApp/app/store/People.js 找到model fetch資料

itemTpl: ‘{Name}{Id}: {Email}’, 定義了輸出資料的方式

List也可以設定event handler 但是叫做listeners

Ext.define('SogetiApp.view.Address',{

    extend: 'Ext.List',
    xtype: 'addresspage',

    config:{
        title: 'Consultants',
        iconCls: 'star',
        store: Ext.create('SogetiApp.store.People'),
        itemTpl: '{Name}{Id}: {Email}',
        listeners: {
            itemswipe: function (list, idx, target, record, evt) {
                Ext.Msg.alert('itemswipe', record.data.name);
            }
        }
    }
});
