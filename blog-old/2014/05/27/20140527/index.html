語系判定

jquery.validate的判定語系方式和使用payload的判定方式不同

java : ResourceBundleMessageSource

一開始在判定要輸出什麼文字在動作完之後,是在前端做判斷,所以national language support(nls)裡面有error.js和message.js,用來做處理多國語言的情況,但是後來Jason認為,將資料傳到後端做處理後,就應該在處理的過程中就知道有是否新增成功,修改成功或失敗等訊息,所以改為文字訊息是從後端發出,也就是payload夾message到前端去做顯示,locale判斷你是處於哪個地區,再來決定是set哪種語言內容

javascript判斷語系

workflow:java spring

controller做完新增 修改 刪除的動作後,用payload set進一些message 或 httpstatus之類的,傳到前端

messageResource.getMessage("0002", null, locale)
會根據locale去抓 src/main/resources/ status_en.properties or status_zn_CN.properties or status.properties(default)

而MessageResource.java又是使用ResourceBundleMessageResource.java的function => getMessage()

set ResourceBundleMessageSource的一些屬性

bean.setFallbackToSystemLocale(false); 預設是true,true表示會去抓啟動這個網頁的server是屬於哪種locale,而我是用Mac當作是Server來啟動此網頁,所以會去抓Mac的語系

問題來了..

我安裝Mac , Chrome時 一開始都是裝中文,但後來調為英文,

但Spring在抓的時候,認為Chrome是中文 送回zh-TW , 但我們預設是status.properties 沒有status_zh_TW.properties,所以系統變成找不到相對應的語言檔,因此被視為無此檔案,使用setFallbackToSystemLocale(true)去抓系統語系,而Mac被視為是已經轉為英文的en_US,所以又被轉回英文,所以在每一次找不到相對應語言檔時,就會使用en_US

設定default locale , 但跟這次碰到的問題,沒有多大關係
