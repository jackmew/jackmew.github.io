link

work

其實vaadin的sample project , 原本就有import , 所以產生出來後 , 就可以馬上mvn package

mvn jetty:run

但是現在要考慮的是:

  1. mvn package or mvn compile 之後 產生的檔案(war or class) 會放到哪去?

  2. jetty去哪個folder找 產生的檔案? , 什麼時候去找?

設定build之後產生的位置

指定的id : development 是命名我們自己想要用的build

因此 這指定 mvn compile -P development 就會去使用這個build 並且build到我們指定的位置(其實這個位置在下mvn package的時候是同樣位置 , 但是package時 多了build war) , 而這個位置就是jetty 預設會去找的位置

jetty scanIntervalSeconds

我覺得最主要的就是這段

<scanIntervalSeconds>1</scanIntervalSeconds>

設定jetty去scan folder內有不同的時間! cool!!!

mvn jetty:run-exploded 這個跟mvn jetty:run差不多? 測試的結果是都可以

enter eclipse

注意在import時候 可以設定development 也就是你build之後的檔案位置 會因此改變

原本是:jettyTest/target

run 設定新的run configuration

而且我發現 一定要是jetty:run-exploded 在ctrl+s 才會自動去build 自動redeploy !

jetty:run vs jetty:run-exploded

link

jetty:run => jetty在run 時 是不需要war檔的 , 可以直接抓.class(也就是source) 就可以直接run , 所以說 如果我們不多一步 package war 就可以減少build時間!

jetty:run-war

jetty:run-exploded => save會自動去build 自動redeploy

其實在官網的說明 其實都說會自動redeploy 但是我測試的結果 只有jetty:run-exploded會

jetty debug

這時候jetty不具備debug能力

remote debugging

一開始的jetty:run-exploded一樣

但是為了enabled Remote Debugging 要設定監聽-Xdebug -Xrunjdwp:transport=dt_socket,address=4000,server=y,suspend=n

這樣子run 就會啟動jetty server , 並且監聽port:4000

然後設定remote java application

就成功了!!!
