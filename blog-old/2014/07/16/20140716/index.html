setInterval or callback問題

call function 時的怪問題

正常執行

setInterval(call , 30);

function call() {
   console.log('setInterval');
}

只會執行一次call()

setInterval(call() , 30);

function call() {
   console.log('setInterval');
}

pass function reference 才沒有問題

setInterval(function(){call()} , 30);

function call() {
   console.log('setInterval');
}

原因:canvas z-index:-1 所以當canvas要抓 mouseover event的時候 會抓不到

所以去找document的mouseover event去抓位置 , 但是這時候又發現 $(document) 沒有parentOffset , 又會出現 當往下拉畫面時 滑鼠的位置出錯問題 , 所以problem 1 的解決方法也不能用

scrollleft

  1. 當沒有給參數時 會回傳當前horizental位置

  2. 當有給參數時 就會設定到那個位置

     $(document).mousemove(function(e){
         mouse.x = e.pageX - lastScrolledLeft;
         mouse.y = e.pageY - lastScrolledTop;
     });
     $(window).scroll(function(e) {
         lastScrolledLeft = $(document).scrollLeft();
         lastScrolledTop = $(document).scrollTop();

     mouse.x -= lastScrolledLeft;
     mouse.y -= lastScrolledTop;
 });

solving : 記錄每次scroll的位移量 然後捲動和移動時都要去扣掉scroll的位移量
