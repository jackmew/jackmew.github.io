#Build Automation Software for Java

Ant -> Maven-> Gradle

#log4j

設定方式有兩種

1.log4j.properties
2.log4j.xml

link

#Maven

mvn archetype:generate

mvn archetype:generate \ -DarchetypeArtifactId=maven-archetype-quickstart \ -DinteractiveMode=false \ -DgroupId=tw.com.codedata -DartifactId=helloworld =>產生基本專案

mvn help:effective-pom => 查詢現在 pom.xml 的有效設定

mvn exec:java => 執行java

mvn exec:java -Dexec.mainClass="tw.com.codedata.App"

我根本就不用手動去下載log4j , 只要在pom.xml裡加上

<dependency>
    <groupId>commons-logging</groupId>
    <artifactId>commons-logging</artifactId>
    <version>1.1.1</version>
</dependency>
<dependency>
    <groupId>log4j</groupId>
    <artifactId>log4j</artifactId>
    <version>1.2.16</version>
</dependency>

下mvn package 後就會自動下載需要的jar

在maven/apache-maven-3.1.0/conf/settings.xml

預設repository放置的位置是~/.m2 , 所以只要你有設定在pom.xml裡面的dependencies, 都會下載到這個位置

#gradle

以build.gradle(DSL) 取代 pom.xml

###gradle plugin

apply plugin: 'java'
apply plugin:'application'

###gradle command

gradle tasks 列出可以執行的task

gradle distZip 產生 -> build/distributions/helloworld.zip

gradle dependenecies -> 顯示相依性

gradle build

gradle run

gradle document

###相依關係設定

compile 'com.amazonaws:aws-java-sdk:1.+'

怎麼來的?

原本有加上apply plugin:’application’

所以可以直接gradle run

但改掉後 就沒辦了

build.gradle:

apply plugin: 'java'

repositories {
    mavenCentral()
}

dependencies {
    compile group: 'commons-logging', name: 'commons-logging', version: '1.1.1'
    compile group: 'log4j', name: 'log4j', version: '1.2.16'
}

sourceSets {
    main {
        java {
            srcDir 'src'
        }
        resources {
            srcDir 'config'
        }
    }
}

gradle build 沒有問題

但是java -classpath builds/lib/helloworld2.jar tw.com.codedata.App 出現lost logFactory的訊息

看一下helloworld2.jar 裡面 沒有包入commons-logging,也沒有包入log.class

需要再加入

jar {
    from configurations.compile.collect { zipTree it }
}

表示會在jar中加入dependecies

java -classpath builds/lib/helloworld2.jar tw.com.codedata.App
