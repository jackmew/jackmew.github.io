##Weinre(Web Inspector Remote)

開發Mobile Web的Remote Debugging工具這篇是用andriod自己的adb工具才做debug

Getting started with Weinre mobile debugging on Windows

ifconfig | grep "inet" 找到ip後

weinre --httpPort 8081 --boundHost 192.168.26.115 設定ip 將預設的port:8080 改成8081

  1. 非常簡單的html頁面

關鍵點是在裡加入

<script src="http://192.168.26.115:8081/target/target-script-min.js#anonymous"></script>

就連上去了

  1. phonegap project

產生最基本的phonegap project

~/jachho/Sites/weinre

phonegap create weinrepg1 com.example.weinrepg1 weinrepg1

發現run的時候會出現這錯誤 這是正常的

Phonegap.js not found in development當下sencha app build native or 單build ios,android後 才會出現phonegap.js

更改~/jackho/Sites/weinre/weinrepg1/www/index.html

head加入<script src="http://192.168.26.115:8081/target/target-script-min.js#anonymous"></script>

body加入<h1 style="color:pink">this is the colorful text</h1>

也成功了

  1. sencha phonegap weinre

產生最基本的sencha project

~/jackho/Sites/weinre/

sencha -sdk ~/sencha/touch-2.3.1/ generate app weinresencha ./weinresencha

cd weinresencha

sencha -sdk ~/sencha/touch-2.3.1/ phonegap init com.example.weinresencha weinresencha

更改~/jackho/Sites/weinre/weinresencha/index.html

一樣加入<script src="http://192.168.26.115:8081/target/target-script-min.js#anonymous"></script>

~/jackho/Sites/weinre/weinresencha/app.js

launch加入console.log("weinre sencha success");

有抓到console

但我發現完全沒辦法對javascript debug…

  1. 將3的project build 成apk 安裝到模擬器

android avd start 模擬器

再次開不起..

finder cmd + shift + g 路徑:~/.android

找到~/.android/avd裡面的avdName.ini avdName.folder 都刪除

再次new一個weinresencha.avd 然後start

adb devices

找到emulator-5554 device attached

~/jackho/Sites/weinresencha

sencha app build -run native

失敗了 server抓不到sencha

更改~/jackho/Sites/weinresencha/config.xml

<access origin="*"/>

再次sencha app build -run native

改變畫面

也可以產生alert()

關掉avd後 又不能開…

terminal cmd + shift + g ~/.andriod/avd/weinresencha.avd/emulator-user.ini

將裡面window.x和window.y的值改成0

就可以再次開啓

  1. 安裝到手機上

先到https://build.phonegap.com/apps刪除已經安裝的app

因為免費賬號 一次只能放一個app

發生奇怪的問題UNABLE_TO_VERIFY_LEAF_SIGNATURE

先直接放到dropbox的public folder裡面

手機點選dropbox一樣可以下載

copy ~/Sites/weinre/weinresencha/phonegap/platforms/android/ant-build/HelloWorld-debug.apk
to

啓動weinre server

weinre --httpPort 8081 --boundHost 192.168.26.115

下載安裝後 在手機上執行app

抓到了!

一樣產生一下alert()看看

成功了！ 超酷！

##繼續JSONP 無法在手機上call的問題

用weinre 在啓動的emulator上的app時 launch的console.log會抓不到

因此我把launch的call ajax jsonp function取消掉 讓botton按到的時候才去call controller上的function

可以看到app.js裡面有抓到controllers:birdCt 而且是在launch之前 所以birdCt的init在launch之前執行

接下來我讓第一頁加上一個button

並在birdCt控制button

當button tap事件發生 call doCallAjaxJsonp
接下來再doCallAjaxJsonp call this.loginJsonp();

接下來loginJsonp success 再call this.getJsonp();

可以看到網頁版app的可以call ajax jsonp 抓到cross-domain的資料

但是install到emulator後失敗

解決方法,因為模擬器不認識localhost

ifconfig | grep "inet"

找到inet 192.168.26.115

將localhost的地方都取代成192.168.26.115

android avd start emulator

adb devices 確定emulator有連接上

~/jackho/project/sencha/bird3

sencha app build -run native 裝進emulator 並直接run

##接下來要真的從資料庫抓到資料

開啓wbase

輸入url

http://localhost:8080/wbase-web/rest/uid/queryOnePage?currBlock=1&condition=ZAAAAA

會找到一筆資料

放到finalList裡面

ObjectMapper將object 轉成json string

然後將中括號移除

最後包在callback裡面

讓getJsonp success後執行getRealData()

jsonp抓回來的是object 轉成json

成功！

將condition: ‘ZAAAAA’ => condition: ‘ZBBBBB’

水喔！

接下來install 到 emulator中試試看

sencha app build -run native

很好！

接下來安裝到手機裡面

點手機上的button

電腦上的meinre server 抓到console.log()

很好！
