mac mavericks

昨天晚上更新過後 之前設定的node,npm,mongo,weinre,express,hexo,node-inspector都出現command not found

好像是把/usr/local/bin/內 hyperlink的部分都清除了 應該是都設定為-g(global) 都會無法直接下 可以經由.bash_profile設定path

所以-g表示的意思是 設定到/usr/local/bin底下

  1. 首先解決node,npm,mongo的方法是設定PATH

  1. weinre,hexo,node-inspector,phonegap npm install -g weinre

npm install -g hexo
npm install -g node-inspector

npm instal -g phonegap

start finder shift+cmd+g path:/usr/local/bin 這個路徑會放置link給設定global的package

  1. express npm install -g express 照理說應該可以直接使用才對 在/usr/local/lib/node_modules/express有個Readme.md
    另外再下npm install -g express-generator@3
    express的link就會出現在/usr/local/bin

這個動作也會同時將需要用到的模組設成global,require才抓的到

其實最好都用一樣的方式來設定比較有統一性?
不過都用 反正可以多理解 也沒差

小整理

用homebrew安裝的package會放在/usr/local/Cellar

用npm安裝的package會放在/usr/local/lib/node_modules 如果在安裝的時候設定-g ex:npm install -g weinre 會在/usr/local/bin 設定link
On npm and Homebrew 會發現npm,homebrew可以
都安裝同一種套件 比如mongodb,然而他們管理套件的方式不一樣,有時後可能發生某種衝突? 像這次安裝express下了-g卻沒有用 還必須看ReadMe裡面多下一條npm install -g express-generator@3 ,所以套件管理到底好不好…? 有點複雜..但還是想用呀！ 用起來實在太酷了！

Installing node.js on OSX 10.9 Mavericks 仔細想想 我根本不需要用到homebrew,只要安裝這個就好拉…就npm就好了
