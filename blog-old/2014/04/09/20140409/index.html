##Node.js 事件迴圈

要達到Asynchronous I/O的特性,必要有一個監聽者的角色來處理非同步發出的請求,像callback開始後 要花一點時間完成 所以先執行了後面的程式,那誰來接收callback回來的結果呢,是因為有個監聽者一直在不斷的迴圈,看有沒有任何事件,請求,回應會突然出現,這監聽者的角色就是 libev

##CommandJS

用來定義Javascript API,規範,模組(module),套件(package)的一種標準

##RequireJS

JavaScript本身就具備Require指令來載入模組,然而我們發現並不是所以模組都要在一開始就載入好的

RequireJs可以做到當下需要的模組再載入的這個想法也就是Asynchrnous Module
Defintion(AMD),並且確保所以相依(dependency)當載入才會執行

CommandJS和RequireJS有什麼不同

##module & package

package比較像是組合一堆module之後的完成品

export : 公開模組

require : 取得模組

有點類似於java的class

//hello.js

function Hello() {
 var name;

 this.setName = function(thyName) {
   name = thyName;
 };

 this.sayHello = function() {
   console.log('Hello ' + name);
 };
};

module.exports = Hello;

有點類似於java new 實體

//gethello.js

var Hello = require('./hello');

hello = new Hello();
hello.setName('BYVoid');
hello.sayHello();

Javascript是函數式程式設計語言,並沒有class概念,function本身就是物件,在node又叫做module,module可以藉由export公開,require引用

##package.json

CommonJS規定用來描述套件(package)的檔案

Node.js會先去找package.json裡面的main 決定要先執行哪個js 若是沒有package.json就會去找index.js or index.node

##npm update npm -g

npm無法自動update自己…

似乎是因為brew install node的時候 就同時安裝了npm

由於brew , npm上安裝模組的方式有異 所以產生某些衝突

elgs提出的方式是
brew install node –without-npm
curl https://www.npmjs.org/install.sh / curl -L https://npmjs.org/install.sh | sh

因為homebrew是套件管理 npm也是套件管理 這樣的做法就讓各自管理自己套件

不過我還得要刪除再重新安裝..

另外的方式

npm install -gf npm => -f是force的意思 好像會有一些link的問題

的確是成功update npm成1.4.6 不知道之後會不會有問題產生…

有些人說加上這句

brew unlink node && brew link –overwrite node

但是我加上去後 就變回原來npm 1.4.3

現在node , npm都是最新版本 之後有問題 就用elgs的方式重新安裝

##Express

為了在Terminal使用很多Quick start 工具 ,我們將Express裝在全域模式下

npm install -g express

安裝後的位置是在/usr/local/lib/node_modules/express

模版引擎1.Jade 2.ejs

使用ejs(Embeded JavaScript)

~/jackho/node/

express -t ejs jackmewNodeBlog

產生~/jackho/node/jackmewNodeBlog

cd jackmewNodeBlog

npm install 它會根據產生的package.json dependency產生需要的檔案

node app.js =>Express server listening on port 3000 in development mode

在browser輸入localhost:3000

我想要關掉node時 按下ctr+Z 顯示stop 但好像還會佔用 導致我後來無法再次開啓node

砍掉的方式

lsof -wni tcp:3000 去找誰佔用了port:3000 記住PID

kill -9 PID

之後就可以再次啓動node app.js or supervisor app.js 後者可以在你改了code後馬上refresh

之後要確實關閉node 按ctr + C
