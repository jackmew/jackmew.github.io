##install mongodb

brew install mongodb

mongod =>出現一些資訊 要設定一些startup options

which mongod => /usr/local/bin/mongod

solution for **WARNING: soft rlimits too low

solution for exception in initAndListen: 10296 dbpath (/data/db) does not exist, terminating :mkdir -p /data/db/ 會放上一個隱藏資料夾在root

solution for http://stackoverflow.com/questions/7948789/mongodb-mongod-complains-that-there-is-no-data-db-folder 這表示那個資料夾權限不夠 sudo chown ``id -u`` /data/db

接下來再次執行mongod 依然會會出現很多資訊 但已經不再有error or exception

發現直接執行mongo會無法開啓mongo shell

solution for Error: couldn’t connect to server 127.0.0.1:27017 (127.0.0.1)

所以最直接的安裝步驟 應該直接照著Ali Emre Çakmakoğlu的回應

brew install mongodb

create main db directory
sudo mkdir -p /data/db

Make sure that you have a read and write permission on that directory
sudo chown ``id -u`` /data/db

isten to default port
mongod

Terminal會顯示listening Cmd + T 再開啓一個terminal

輸入

mongo 就可以正常開啓mongo shell…

##How express manipulate mongo?

接下來找到底express跟mongo到底是怎麼合在一起操作的?

想想之前Java在Spring+Hibernate的架構下 Hibernate設定好 準備好JavaBean 接著就可以操作資料

那express跟mongo連接的那一層是什麼?

一堆資訊都在這:
mongoDB Node.js Language Center

node-mongodb-native 有實際demo的code

The MongoDB Node.js driver is the officially supported node.js driver for MongoDB. 這個driver是由Christian Kvalheim寫的node-mongodb-native ,後來mongo跟他一起合作?

這個driver也可以當成底層來support實現object mapping,比如Mongoose(ODM :Object-Document_Mapper)

還有mongoose tutorial

npm install mongoose

我用homebrew安裝mongodb,brew install mongodb

不知道跟用npm安裝會不會不一樣….npm install mongodb

因為之後mongoose也是用npm install mongoose 路徑上的設定不知道會不會有問題?!

應該說是NodeJS跟Mongo中間有個實現的driver,Express只是個架構?

而且為了實現NodeJS+Mongo 不止require mongodb 還有require connect-mongo

##node-inspector

我發現我好像不能在node server那邊debug

還是要裝node-inspector

sudo npm install node-inspector

啓動node-inspector

cmd + t開啓另一個terminal 輸入 node-inspector

顯示
Node Inspector v0.7.3
Visit http://127.0.0.1:8080/debug?port=5858 to start debugging.

在另一個terminal啓動你的app

cd到你要啓動的app位置

node --debug app.js

node --debug-brk app.js 一開始就停住app
