MongoDB initialized

MongoDB

http://www.mongodb.org/

mongo => Document-Oriented database

mysql,ms-sql => Relational-Oriented database

mongoDB的使用上,不需要先建立儲存資料的結構,定義資料型別,基本上你可以把所有key:value格式的資料丟進去,ex:JSON,Array,Hash,非常適合物件導向

不用先create資料表 在mongoDB只要找不到對應的collection就直接新增一個collection

想必大部份人都比較熟關聯式資料庫,所以看表格來學習,我覺得是比較快速的

http://docs.mongodb.org/manual/reference/sql-comparison/

SQL mongo
Relational-Oriented Docment-Orented
table collection(channel)
row document

Mongo GUI => Robomongo

一个基于 Shell 的跨平台开源 MongoDB 管理工具 嵌入了 JavaScript 引擎和 MongoDB

右上的黑色部分 輸入javascript語法 點擊左上的綠色三角形 執行

MongoLab - mongoDB Hosting:Database-as-a-service

https://mongolab.com/

因為Mac的硬碟空間已經很小了,不想要再裝DB在裡面,就試著想了個免費的database hosting試試看,沒想到非常簡單

註冊之後 找到這個頁面

按下 create new

plan選擇development就可以免費使用500mb

成功後 點一下就可以看到URI

接著會叫你add new database user

之後就會給你連結此資料庫的資訊

在Robomongo create new connection
填上Address,port,database user

Done

my first post 'hexo'

Hexo

node.js blog framework

http://zespia.tw/hexo/

Install

npm install -g hexo

cd hexo_JackMew/
(the directory you want to put hexo file)

hexo init

then you’ll see these file in the directory.

_config.yml,package.json,scaffolds,scripts,source,themes

hexo generate

hexo server

deploy

vim _config.yml

deploy:
    type:` `github
    ropository:` `https://github.com/yourname/yourname.github.io.git
    bracnch:` `master

the point is that you need to have blank after :

hexo generate

hexo deploy




new post




hexo new "your post name"

cd source/_posts

you’ll see the new file postname.md

vim postname.md

Categories and tags will create link to your post.

When you finished the post.

hexo generate again

hexo deploy

The new post is on the blog now , Amazing ! how easy it is !!!
