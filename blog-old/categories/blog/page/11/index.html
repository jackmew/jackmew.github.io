shell

但是以上這些動作有些複雜 而且是每次都重複的動作

所以我寫了個shell script來執行這些動作

#!/bin/bash

#clear ~/hexo_Jackmew , and transform markdown to html
cd ~/hexo_JackMew
hexo generate

#move original index.html to new folder /page/1
mkdir ~/hexo_JackMew/public/page/1
mv ~/hexo_JackMew/public/index.html ~/hexo_JackMew/public/page/1

#copy my personal web site to the public folder , and it has index.html
cp -r ~/hexo_JackMew/JMC_me/dist/ ~/hexo_JackMew/public/

#deploy
hexo deploy

就可以用terminal來快速完成啦～

Requesting a file system

    start(){
        window.requestFileSystem(window.TEMPORARY, 5*1024*1024 /*5MB*/, onInitFs, errorHandler);
    }

    function onInitFs(fs) {
      console.log('Opened file system: ' + fs.name);
    }
    function errorHandler(e) {
      var msg = '';

      switch (e.code) {
        case FileError.QUOTA_EXCEEDED_ERR:
          msg = 'QUOTA_EXCEEDED_ERR';
          break;
        case FileError.NOT_FOUND_ERR:
          msg = 'NOT_FOUND_ERR';
          break;
        case FileError.SECURITY_ERR:
          msg = 'SECURITY_ERR';
          break;
        case FileError.INVALID_MODIFICATION_ERR:
          msg = 'INVALID_MODIFICATION_ERR';
          break;
        case FileError.INVALID_STATE_ERR:
          msg = 'INVALID_STATE_ERR';
          break;
        default:
          msg = 'Unknown Error';
          break;
      };

      console.log('Error: ' + msg);
    }

首先因為放大縮小的問題 我將px改成em

em is scalable and px is not. Set the font to a defined px size and you should be ok. em can be desirable in certain circumstances, but if you are worried about 1px then you should set strict pixel sizes.

em是根據font size大小作為單位,然而chrome跟firefox的font size單位卻不一樣…

再加上difference-size-font-rendering-on-windows-vs-macos-how-to-solve,所以不同台電腦上也可能會顯示不同的font size,所以結論是不應該用em來定義top , margin , padding等 , 用px會比較嚴謹 , 然而又很難處理放大縮小問題.
