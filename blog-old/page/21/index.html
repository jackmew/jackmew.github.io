heroku

###PaaS

在那時候我還不是很了解hosting是什麼意思,就覺得heroku界面很酷,好像用了就會很屌,所以就去試試看.原來這就是之前唸書時常聽到的PaaS(Platform-as-a-Service),在Heroku上是比較配合Ruby的開發,但官方還是有提供java和node的開發啦.

幾種我聽過的

  • Phpfog => php
  • Heroku => ruby
  • OpenShift => java Linux
  • GAE(google App Engine) => java

其實都可互相通用?

這些平台的目的 減少心思投入在系統管理,架構 專注在應用程式開發上,但話說回來 要熟悉使用某平台 本身就要改變自己熟悉的framework還有版本控制等習慣,而我試用的狀況下server是沒用過的Jetty.

不過還是覺得很厲害,像在現在的公司,也是老闆構思專案架構,將workflow定義好,其中細節再交給我們去做,因為所謂架構(framework)是一連串經驗的集合,經驗不足時只有在做到某個部分才會理解,oh~原來是因為xxxx….才要….

MVC,一堆Design pattern 都是驗經累積的成果,因為往往在專案還是很小的時候不會發生問題,但專案一變大,又不是一人開發,就會碰到很多問題,也是在這時候才徹底理解為何要

所以平台的彈性和資源若是提供的不夠充分,使用起來綁手綁腳就會讓人不想要使用咧.

很容易就可以在local和server端啓動程式 rocky-forest

###SpringMVC

一直無法在local端執行

直到找到rocky-forest-1180底下的readme

first build with:

$ mvn clean install 常用 可以理解

Then run it with:

$ java -jar target/dependency/webapp-runner.jar target/*.war 這裡跟我們用tomcat deploy的時候差很多 不是很能理解 但後來在Spring的教學又看到類似的東西 就有去研究一下

稍微統整一下 建置一個環境要做的事情

一開始svn checkout source code | git clone source code

基本上裡面都會包含一些設定檔xml , pom.xml | procfile

接下來要看你有沒有安裝正確的java版本 maven版本tomcat|jetty
安裝好以後 路徑也ok就可以成功了

但那是在command line執行成功

不代表你在eclipse的build環境也架構好了 eclipse裡面也有git 叫做Egit

而我們開發的專案不止一個 但enviromental variables卻是不同的

所以會有一個setenv.sh檔案 裡面set(export) enviromental variables

之前Heroku java是建立了heroku的專案 會自動配一個database

若是你建立了blank的專案 就必須要手動加入一個database 在heroku中database屬於add-ons的其中之一 當你deploy專案上heroku之後 你可以點選add-ons加上 postgreSQL database ,建立之後 點選資料庫 就會出現detail

在rocky-forest-1180/target/classes/applicationContext.xml設定
並加入

<property name="connectionProperties" value="ssl=true;sslfactory=org.postgresql.ssl.NonValidatingFactory;"/>

我們在local端執行就可以抓到database資料了!

###programming language

當接觸到Wordpress時,發現它是由PHP建立的,之前也很常聽到PHP,因此馬上去google了一下語言的比較,到底Java好還是PHP好,

但google後才發現還有Ruby ,Python,甚至現在Javascript都已經非常強,原本只能當Client-side的語言,有了NodeJS之後還能做Server-side,那不是Javascript最強?!

在搞不清楚之下,先建立了PHP環境,沒想到mySQL一開始就設定錯誤,花了點時間

這裡是拼戰的血淚史php apache mysql

設定好之後,由於Mac就已經內建Apache,所以開啓server很方便,其實php也不太需要重啓server.

這時候發現PHP完全不用像java一樣一直重複build,restart server -> build,restart server…這麻煩的動作,save refresh馬上就可以看到結果!真是太方便了,

但工作需要,還是繼續寫Java,當程式變的龐大後,更理解為何要encapsulation,module ,就覺得其實Java也不錯嘛

這篇寫的有道理php-vs-java

其實每種語言都有自己的優缺點,能夠存活下來的語言肯定是夠強的,然而我們往往還不夠深入,不夠努力 無法理解那背後的道理

而且這之後又各有framework,還不止一種

Ruby => Ruby on Rails

Java => Spring,Hibernate

Python => Django

PHP => codeigniter

Javascript => Express

所以啊..就繼續努力的一直鑽下去吧!!!

不過還是覺得build,restart server很煩 ,在sublime & liveReload 寫js時,cmd + s馬上看到更改結果,就是爽~

有一句讓我印象深刻的話 以大多数人的努力程度之低，根本轮不到去拼天赋。 戒慎恐懼

往往認真多花一點時間 程式都是寫得出來的!

努力鑽!

####query efficiency

@Override
public String getUIDUserString(List<String> userUIDList, String condition,String functionName){

        String startStr ="";
        String lastStr ="";
        if(functionName.equals("queryOnePage")){

            startStr = "{ $query:{$and:[{uid:{$regex:'^"+condition+"'}},{deleteMark:'N'},{$or:[";
            lastStr =" ]} ]},$orderby:{uid:1} }";

        }else if(functionName.equals("queryCount")){
            startStr = "{ $and:[{uid:{$regex:'^"+condition+"'}},{deleteMark:'N'},{$or:[";
            lastStr =" ]} ]}";
        }
        StringBuilder sb = new StringBuilder();
        sb.append(startStr);
        for(int i=0;i<userUIDList.size();i++){
            sb.append("{uid:{$regex:'^"+userUIDList.get(i)+"'}}");
            if(i != (userUIDList.size()-1)){
                sb.append(",");
            }
        }
        sb.append(lastStr);
        String queryStr = sb.toString();
        return queryStr;
    }
