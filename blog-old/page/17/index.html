##CSV vs XML vs JSON

CSV(Comma-separated values) : 傳回的資料是用逗號分隔，文字內容由雙引號夾住，一行代表一筆資料的格式。excel可以匯入csv

這樣子的定義還蠻像JSON的

實際比較

CSV Comma-separated value

優點 : 資料壓縮最小的格式

缺點 : 不支援多層(hierarchy)結構

XML extensible markup language

優點 : W3C的標準格式 很好的支援多層結構

缺點 : 包了很多tag , size比CSV大三倍

JSON

優點 : 支援多層結構 object literal的方式很適合變成包成javaBean那種object

缺點 : size比CSV大兩倍(比XML小)

結論 JSON最適合從server傳資料到client！

###mac mavericks

昨天晚上更新過後 之前設定的node,npm,mongo,weinre,express,hexo,node-inspector都出現command not found

好像是把/usr/local/bin/內 hyperlink的部分都清除了 應該是都設定為-g(global) 都會無法直接下 可以經由.bash_profile設定path

所以-g表示的意思是 設定到/usr/local/bin底下

  1. 首先解決node,npm,mongo的方法是設定PATH

  1. weinre,hexo,node-inspector,phonegap npm install -g weinre

npm install -g hexo
npm install -g node-inspector

npm instal -g phonegap

start finder shift+cmd+g path:/usr/local/bin 這個路徑會放置link給設定global的package

  1. express npm install -g express 照理說應該可以直接使用才對 在/usr/local/lib/node_modules/express有個Readme.md
    另外再下npm install -g express-generator@3
    express的link就會出現在/usr/local/bin

這個動作也會同時將需要用到的模組設成global,require才抓的到

其實最好都用一樣的方式來設定比較有統一性?
不過都用 反正可以多理解 也沒差

####小整理

用homebrew安裝的package會放在/usr/local/Cellar

用npm安裝的package會放在/usr/local/lib/node_modules 如果在安裝的時候設定-g ex:npm install -g weinre 會在/usr/local/bin 設定link
On npm and Homebrew 會發現npm,homebrew可以
都安裝同一種套件 比如mongodb,然而他們管理套件的方式不一樣,有時後可能發生某種衝突? 像這次安裝express下了-g卻沒有用 還必須看ReadMe裡面多下一條npm install -g express-generator@3 ,所以套件管理到底好不好…? 有點複雜..但還是想用呀！ 用起來實在太酷了！

Installing node.js on OSX 10.9 Mavericks 仔細想想 我根本不需要用到homebrew,只要安裝這個就好拉…就npm就好了

可是這樣就要重新new一次avd了..

google一下how to close avd 才發現大家都有這問題…到底..

稍微解釋

或許只要開啓一次就要刪除一次..

只要確實在termial上按下ctrl+c關閉avd就可以確實關閉

terminal

android avd prompt GUI 手動start avd

terminal cmd+t 開啓另外一個termial

adb devices 會看到你start avd有attached到

找到你要安裝的apk路徑 我的例子是在~/phonegap/hello/platforms/android/bin/HelloWorld-debug.apk

我先cd ~/phonegap/hello/platforms/android/bin

adb -s emulator-5554 install HelloWorld-debug.apk

此時就可以在emulator上看到你的app了

此時環境已經確定都ok了

再次重新做一次phonegap project到emulator

##phonegap project install to emulator

~/phonegap

phonegap create pg com.example.pg PGProject

產生最基本的phonegap project

cd pg/

phonegap build android產生支援android project

cd platforms/android/

ant debug 編譯android project

產生要安裝的apk在~/phonegap/pg3/platforms/android/bin/HelloWorld-debug.apk

打開emulator

android avd start 你要啓動的emulator

cmd + t 開啓新的terminal

adb devices

output:

List of devices attached
emulator-5554 device

找到emulator名稱

adb -s emulator-5554 install ~/phonegap/pg3/platforms/android/bin/HelloWorld-debug.apk

##sencha to phonegap to emulator

cd phonegap

sencha -sdk ~/sencha/touch-2.3.1/ generate app spg ./spg

產生好最基本的sencha project

cd spg/

sencha -sdk ~/sencha/touch-2.3.1/ phonegap init com.sencha spg 包成phonegap

修改
spg/phonegap.local.properties

phonegap.platform可以決定你要build成哪種平台

phonegap.build.remote決定要不要遠端build並放上phonegap.build雲端上

你註冊的賬號密碼
phonegap.build.remote.username
phonegap.build.password

sencha app build -run native

我是在已經啓動android avd的情況下 下達這句指令 然後就直接安裝並啓動sencha app了！

所以應該是先啓動android avd再做之後build的動作

在build的時候產生了~/phonegap/spg/phonegap/platforms/android/ant-build/HelloWorld-debug.apk

adb -s emulator-5554 logcat 看emulator的log

但裡面超多log的 很不清楚…

##實作Weinre

Java compile後變成war ,deploy到tomcat上,雖然已經被包在war裡面了,還是可以debug

所以雖然sencha最後被包成apk,當然也可以debug

安裝weinre

npm -g install weinre

weinre --boundHost -all- 這樣就可以啓動debug server
