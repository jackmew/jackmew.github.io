hexo

hexo 是用來記錄的blog產生模組

所以我在另外的資料夾建立一個JMC_me 的web project

寫好後 將web project 轉移到hexo_JackMew資料夾

將原本的index.html移動到page folder

用web project的index.html取代

shell

但是以上這些動作有些複雜 而且是每次都重複的動作

所以我寫了個shell script來執行這些動作

#!/bin/bash

#clear ~/hexo_Jackmew , and transform markdown to html
cd ~/hexo_JackMew
hexo generate

#move original index.html to new folder /page/1
mkdir ~/hexo_JackMew/public/page/1
mv ~/hexo_JackMew/public/index.html ~/hexo_JackMew/public/page/1

#copy my personal web site to the public folder , and it has index.html
cp -r ~/hexo_JackMew/JMC_me/dist/ ~/hexo_JackMew/public/

#deploy
hexo deploy

就可以用terminal來快速完成啦～

其實在普通的html 只要加上這一行 就可以了

但是若是一整個web , 需要開放給外面抓的到,才可以使用

而在java springMVC 的開放方式是

首先/WEB-INF/這個folder是不開放的,應該說基本上所有程式都不對外開放,而/WEB-INF/擁有開放的功能,所以我們將/dhtml/對應任一個folder or 檔案 的位置 都dispatch到/WEB-INF/dhtml/ , 並開放/WEB-INF/dhtml/ 這個資料夾, 所以任何人指定路徑到這個folder底下 都可以用瀏覽器抓到.
