##Passing Route Control

路由規則(Route Control)

  1. 同一條規則(同一個function宣告or使用) 會只用先比對到的function

    app.get('/users/:id?', function(req, res, next){
    var id = req.params.id;
    if (id) {
        // do something
    } else {
        next();
    }
});

app.get('/users', function(req, res){
    // do something else
});

function的部分可以:

one : 兩個參數 => function(req,res) =>不產生callback next()

two : 三個參數 => function(req,res,next) =>有callback next() 可以passing route control

three : 四個參數 => function(err,req,res,next) =>特別處理errorHandling使用

##module & package

package比較像是組合一堆module之後的完成品

export : 公開模組

require : 取得模組

有點類似於java的class

//hello.js

function Hello() {
 var name;

 this.setName = function(thyName) {
   name = thyName;
 };

 this.sayHello = function() {
   console.log('Hello ' + name);
 };
};

module.exports = Hello;

有點類似於java new 實體

//gethello.js

var Hello = require('./hello');

hello = new Hello();
hello.setName('BYVoid');
hello.sayHello();

Javascript是函數式程式設計語言,並沒有class概念,function本身就是物件,在node又叫做module,module可以藉由export公開,require引用
